# Laurandy — Game Studio

## What this is

A collection of small browser-based games built by Laurent and Andy for fun, learning, and (eventually) passive income. Each game lives in its own directory with its own CLAUDE.md spec.

## Project structure

```
/
├── CLAUDE.md              # this file — project-wide guide
├── docs/                  # shared documentation, notes, decisions
├── Word Chain/            # word association chain game (first prototype, built)
├── skeleton/              # alternating-letters word puzzle (to build)
├── crossfire/             # compound word bridge puzzle (to build)
└── slide/                 # sliding rows vertical word puzzle (to build)
```

## Philosophy

- **Validation prototypes**: each game is a React web app meant to test whether the mechanic is fun. No auth, no monetization, no mobile-specific code. If a game proves fun and gets traction, a Flutter mobile version comes later.
- **LLM-generated content**: puzzle content should be designed so it can be batch-generated by LLMs and validated programmatically.
- **Simple stack**: React + Vite, minimal dependencies. No UI libraries, no state management beyond React hooks, no backend unless needed for validation.

## Conventions

- Each game has its own `CLAUDE.md` describing the mechanic, UI, and build spec
- Game-specific docs go in `<game>/docs/`, shared docs in `docs/`
- Keep prototypes small — the goal is to answer "is this fun?" not to ship a product

## Running a game

```bash
cd <game-folder>
npm install
npm run dev
```
