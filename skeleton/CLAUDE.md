# Skeleton — Validation Prototype

## Purpose

Test whether an alternating-letters word guessing mechanic is fun. The one question this prototype must answer: **Do players enjoy deducing a word from every-other-letter hints?**

## The Mechanic

The player sees a word with every other letter revealed:

```
_ A _ E _ T
```

They must guess the complete word. One guess at a time, 5 attempts max.

- On a correct guess: full word is revealed, show success state
- On a wrong guess: show the guess with correct/incorrect letter highlighting (like Wordle — green for right letter right spot, yellow for right letter wrong spot, grey for wrong letter), but only for the hidden positions since revealed positions are always correct

## Word list

Hardcode 50–100 common English words, 5–8 letters each. Export as an array from `data/words.js`. Pick a random word each session.

For each word, the skeleton is generated by revealing letters at even indices (0, 2, 4, ...) and hiding odd indices (1, 3, 5, ...). Example: `PARENT` → `P_R_N_` or `_A_E_T` (alternate which set is hidden — pick one pattern and stick with it).

## UI layout

- Dark background, centered content
- Title: "SKELETON"
- The skeleton word displayed in large monospace tiles (revealed letters white, blanks as underscores)
- Below: text input for typing a guess + submit button
- Below: list of previous guesses with letter-by-letter color feedback
- Below: "New word" button after win/loss
- Guess count indicator: "Guess 2 of 5"

## Key rules

- Input should auto-focus and submit on Enter
- Guess must be the exact length of the target word (reject otherwise with a shake or message)
- After 5 wrong guesses: reveal the word, show "better luck next time"
- No dictionary validation needed — accept any string of the right length
- Pick a new random word on "New word" click
- Save nothing to localStorage — stateless prototype

## What NOT to build

- No streak tracking
- No daily puzzle mode
- No share card
- No animations beyond basic transitions
- No touch/mobile-specific handling
- No external dependencies (no UI libraries)
